Apa itu Angular?

AngularJs adalah Desain kerja struktural untuk aplikasi-jaringan dinamis. AngularJs mendorong anda menggunakan HTML sebagai bahasa templat dan memperluas sintaksis HTML untuk mengekspresikan komponen aplikasi anda lebih jelas, ringkas dan elegan. Secara intuitif (out of the box), AngularJs menyingkirkan banyak sandi yang (seharusnya) anda buat, melalui data-terikat dan injeksi dependensi. Dan semua proses itu berlangsung dalam peramban, membuatnya sebagai kawan ideal dengan teknologi (sisi-) server apapun.

Angular ibarat HTML yang sesungguhnya jika HTML didesain sebagai sebuah aplikasi. HTML memang hebat sebagai bahasa deklaratif untuk dokumen statik. Namun, HTML sangat sangat kurang, dalam hal membuat aplikasi, dan akibatnya membangun aplikasi-jaringan seperti menjadi ajang latihan tubi dalam arti Apa yang sebaiknya saya lakukan untuk menyiasati peramban mengerjakan apa yang saya inginkan

Hambatan yang tak-imbang antara aplikasi dinamis dan dokumen statik seringkali diselesaikan dengan:

  • Pustaka (library) - adalah koleksi pustaka berisi fungsi-fungsi dimana berguna untuk menulis aplikasi-jaringan. Sandi anda berwenang penuh dan memanggil pustaka bila berasa perlu. msl: jQuery
  • Desain kerja (frameworks) - adalah penerapan khusus aplikasi-jaringan dimana sandi anda mengisi penerapan detilnya. Desain kerja berwenang penuh dan memanggil sandi anda bila memerlukan sesuatu yang spesifik-aplikasi (app specific). msl: backbone, ember, dll.

Angular mengambil pendekatan lain. Angular coba mengurangi hambatan tak-imbang antara pusar-dokumen (document centric) HTML dan apa yang aplikasi butuhkan dengan membuat pembangun HTML baru. Angular mengajari peramban sintaksis baru melalui sebuah pembangun (construct) yang kita sebut direktif. Contoh-contoh yang termasuk antara lain:

  • Data terikat, seperti dalam {{}}.
  • Struktur kendali MOD untuk pengulangan/penyembunyian fragmen MOD.
  • Mendukung isian dan pengesahan isian.
  • Pengikatan sandi-balik (code-behind) pada elemen MOD.
  • Pengelompokan HTML ke dalam komponen pakai-ulang.

Sebuah solusi lengkap untuk sisi-klien

Angular bukanlah potongan tunggal dalam keseluruhan teka-teki pengembangan aplikasi-jaringan-sisi-klien. Angular menanggung semua sandi perekat MOD dan AJAX yang anda tulis dan meletakkannya dalam struktur yang berdefinisi-layak (well-defined). Ini membuat Angular terasa dogmatis mengenai bagaimana seharusnya sebuah aplikasi CRUD dibuat. Namun sementara terasa dogmatis, Angular juga berusaha memastikan bahwa pendapat tersebut hanyalah sebagai titik tolak yang mudah anda ubah. Angular memperlengkapinya dengan cara intuitif berikut:

  • Segala hal yang anda butuhkan untuk membangun sebuah aplikasi CRUD dalam perangkat yang kohesif: data-terikat, direktif templat dasar, ke-sah-an isian, perutean, pertautan-dalam (deep-linking), komponen pakai-ulang, injeksi dependensi.
  • Testabilitas cerita: unit-pengujian, pengujian ujung-ke-ujung (end-to-end/E2E), samaran (mocks) , uji mendayagunakan (test harness).
  • Aplikasi bibit dengan tata-letak direktori dan uji skrip sebagai titik awal.

"Posisi Enak" Angular (Angular Sweet Spot)

Angular menyederhanakan pengembangan aplikasi dengan memberikan abstraksi tingkat tinggi ke pengembang. Dan sama halnya dengan abstraksi apapun, Angular (juga) tidak lepas dari biaya fleksibilitas. Dengan kata lain tidak semua aplikasi cocok untuk Angular. Angular dirancang sebagai aplikasi CRUD sejak awal. Untungnya aplikasi CRUD mewakili mayoritas aplikasi-jaringan. Dengan memahami apa yang Angular mampu lakukan, bagaimanapun, hal itu membantu memahami kapan sebuah aplikasi tidak cocok untuk Angular.

Permainan dan Antarmuka Pengguna Grafis (GUI ) adalah contoh-contoh aplikasi dengan rekayasa MOD yang pelik (tricky) dan intensif. Jenis aplikasi seperti ini memang berbeda dari aplikasi CRUD, dan akhirnya pasti tidak sesuai dengan Angular. Dalam kasus seperti ini lebih baik menggunakan pustaka dengan abstraksi tingkat rendah, semacam jQuery.

Angular Zen

Angular dibangun dengan keyakinan bahwa sandi deklaratif lebih baik daripada imperatif ketika berurusan dengan pengembangan AntarMuka (UI) dan penghubungan komponen peranti lunak bersama, sedangkan, sandi imperatif sangat bagus untuk pengekspresian logika usaha.

  • Adalah sangat baik lakukan pengawagandengan (decoupling) rekayasa MOD dari logika aplikasi. Ini memperbaiki testabilitas sandi secara drastis.
  • Adalah sungguh, sungguh ide bagus menganggap pengujian aplikasi sama penting dengan penulisan aplikasi. Kesulitan pengujian dipengaruhi secara drastis oleh bagaimana sandi terstruktur.
  • Adalah ide brilian untuk mengawagandengkan sisi-klien aplikasi dengan sisi-server. Ini mengizinkan upaya pengembangan melaju secara paralel, dan membolehkan kedua sisi untuk pakai-ulang.
  • Adalah sangat membantu, sungguhan , jika desain kerja (dapat) membimbing para pengembang melalui seluruh perjalanan membangun sebuah aplikasi, mulai pendesainan antarmuka, kemudian penulisan logika-usaha, sampai pengujian.
  • Adalah senantiasa baik membuat tugas umum menjadi sepele dan tugas yang sulit dapat terwujud.

Angular membebaskan anda dari ketidaknyamanan berikut:

  • Pendaftaran balik-panggil (callbacks): Pendaftaran balik-panggil memberantakkan sandi anda, membuatnya sulit melihat hutan dari banyak pohonan. Membuang sandi baku-templat (boilerplate) umum seperti balik-panggil adalah hal bagus. Angular mengurangi secara besar-besaran jumlah penyandian Javaskrip yang seharusnya anda lakukan, dan membuatnya lebih mudah untuk melihat apa yang aplikasi kerjakan.
  • Perekayasaan MOD HTML secara pemrograman: Perekayasaan MOD HTML adalah penyusun utama dari aplikasi AJAX, namun tidak praktis dan gampang-salah (error-prone). Dengan menggambarkan secara deklaratif bagaimana antarmuka seharusnya berubah saat situasi aplikasi berubah, anda dibebaskan dari tugas rekayasa MOD tingkat rendah. Kebanyakan aplikasi yang tertulis dengan Angular tidak pernah harus merekayasa MOD secara pemrograman, walaupun anda bisa jika ingin.
  • Perakitan data dari dan untuk antarmuka: Operasi CRUD memoles mayoritas tugas aplikasi AJAX. Aliran perakitan data dari server sampai objek internal, lalu isian HTML, mengizinkan pengguna mengubah isian, pengesahan isian, penampilan galat ke-sah-an, pengembalian ke model internal, kemudian balik ke server, membuat banyak sekali sandi baku-templat. Angular menghilangkan hampir seluruh hal ini, hanya tinggal sandi yang menggambarkan aliran keseluruhan aplikasi alih-alih semua detil penerapan.
  • Penulisan ton-an sandi inisialisasi hanya untuk mulai: Anda, tipikalnya, perlu menulis banyak sekali perpipaan hanya untuk membuat aplikasi AJAX dasar "Halo Dunia" bekerja. Dengan Angular anda dapat mem-butstrap-kan aplikasi anda dengan mudah menggunakan layanan, dimana otomatis terinjeksi ke dalam aplikasi seperti gaya injeksi dependensi Guice. Ini membuat anda mulai mengembangkan fitur dengan lebih cepat. Sebagai bonus, anda mendapat kendali penuh atas proses inisialisasi dalam uji pengotomatisan.
